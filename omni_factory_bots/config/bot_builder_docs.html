OmniFactory Evo
Core Engine Automation Documentation

Техническая справка по архитектурной организации узла конструктора ботов (Bot Builder Pipeline). Документ регламентирует сквозное взаимодействие между пользовательским интерфейсом управления и бэкенд-модулями компиляции исполняемого кода.

Компоненты и Схема Конвейера
Компонент / Файл Архитектурная Роль и Функционал
bot_modules.ts Фронтенд-реестр конфигураций.
Содержит полную спецификацию из 51 доступного функционального модуля. Хранит метаданные, иконки, категории и текстовые описания блоков управления (включая welcome, voice_to_text, pay_stars, logger, tracking). Напрямую формирует интерактивную матрицу выбора в панели администратора.
builder.py Маршрутизатор API шлюза (FastAPI).
Управляет точкой входа /api/builder/generate. Обеспечивает безопасный перехват POST-запросов от веб-интерфейса, принимает валидированный массив идентификаторов выбранных модулей и пользовательский API-токен целевого Telegram-бота для инжекции в среду.
bot_factory.py Ядро кодогенерации логики.
Выступает фабрикой сборки исходного кода. На основе сопоставления переданных ID компилирует чистый, структурированный код центрального файла запуска main.py, используя асинхронный фреймворк aiogram v3.
project_generator.py Автономный инкапсулятор сред деплоя.
Принимает сгенерированный код, анализирует дерево зависимостей и динамически собирает соответствующий requirements.txt (автоматически подключает langchain и chromadb при активации ИИ-модулей). Формирует конфигурационный Dockerfile и упаковывает готовую микросервисную структуру в ZIP-архив для мгновенного развертывания.
⚡ Интеллектуальный триггер автоматизации

Конвейер спроектирован по принципу изолированной сборки "на лету" в буфере ОЗУ (BytesIO). Это исключает замусоривание дискового пространства сервера временными файлами и гарантирует мгновенную отдачу бинарного потока zip-архива клиенту.
